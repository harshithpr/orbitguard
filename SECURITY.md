# Security Policy

## Supported Versions

OrbitGuard is currently in active development. Security updates are applied to the latest version of the `main` branch.

| Version | Supported |
| ------- | --------- |
| Latest main branch | ✅ |
| Older commits | ❌ |

## Reporting a Vulnerability

If you discover a security issue in OrbitGuard, please report it responsibly.

Please do **not** publicly post the vulnerability in Issues, Discussions, or Pull Requests.

Instead, report it by contacting the maintainer:

**Email:** harshith.praveen16@gmail.com

When reporting, please include:

- A clear description of the issue
- Steps to reproduce it
- Any affected pages, files, or features
- Screenshots or logs if helpful
- The possible impact of the vulnerability

## Response Timeline

I will try to review security reports as soon as possible.

Expected response process:

1. Confirm the report was received
2. Investigate the issue
3. Fix or reduce the risk if valid
4. Credit the reporter if appropriate and requested

## Scope

Security reports may include:

- Exposed API keys or secrets
- Authentication or authorization problems
- Cross-site scripting issues
- Unsafe data handling
- Dependency vulnerabilities
- Deployment or configuration risks

## Out of Scope

The following are usually not considered security vulnerabilities:

- General feature requests
- UI bugs with no security impact
- Broken links
- Performance complaints
- Issues caused by unsupported browsers or outdated devices

## Responsible Disclosure

Please give the project maintainer reasonable time to fix confirmed issues before sharing details publicly.

Thank you for helping keep OrbitGuard safe.
