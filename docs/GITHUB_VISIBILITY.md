# OrbitGuard GitHub Visibility Checklist

Use this after every major OrbitGuard release so GitHub, social links, and search results look polished.

## Repository About Section

Open the OrbitGuard repository, then choose the gear icon beside **About**.

Use this description:

```text
A space sustainability web app that visualizes orbital congestion, simulates satellite launches, and analyzes debris risk.
```

Use this website:

```text
https://orbitguard.vercel.app
```

Use accurate topics:

```text
space
aerospace
satellites
orbital-mechanics
visualization
threejs
space-debris
engineering
javascript
vercel
sustainability
data-visualization
```

OrbitGuard is not a Next.js or TypeScript project right now, so do not tag it as `nextjs` or `typescript` unless the stack changes.

## Social Preview

Go to:

```text
Settings -> Social preview
```

Upload:

```text
assets/social-preview.png
```

This is the same branded preview used by the deployed site for OpenGraph and Twitter/X cards.

## Profile Pins

Go to:

```text
GitHub profile -> Customize pins
```

Pin:

```text
OrbitGuard
AlphaForge
```

Put OrbitGuard first if aerospace is the main story you want people to notice.

## Google Indexing

Go to Google Search Console and submit:

```text
https://orbitguard.vercel.app/sitemap.xml
```

The repository includes `robots.txt` and `sitemap.xml` so crawlers have a clean entry point.

## Sharing Copy

Use this short post when sharing for feedback:

```text
I built OrbitGuard, a space sustainability web app that visualizes orbital congestion, simulates satellite launch impact, and analyzes debris risk using public orbital data. I would love feedback on the aerospace modeling, UI, and roadmap.

Live demo: https://orbitguard.vercel.app
GitHub: https://github.com/harshithpr/orbitguard
```

Good places to share:

```text
r/space
r/aerospace
r/spaceengineering
r/webdev
r/SideProject
LinkedIn
school STEM clubs
aerospace Discords
```

Do not spam. Share progress, visuals, and specific questions people can respond to.
