# OrbitGuard GitHub Launch Plan

Use this checklist to make OrbitGuard easier to discover, star, and share.

## 1. Set The Repo About Box

GitHub repo description:

```text
A space sustainability web app that visualizes orbital congestion, simulates satellite launches, and analyzes debris risk.
```

Website:

```text
https://orbitguard.vercel.app
```

Topics:

```text
aerospace
space
satellites
space-debris
orbital-mechanics
data-visualization
javascript
vercel
sustainability
student-project
```

## 2. Create Starter Issues

Create issues with clear titles so the project looks active and has a roadmap:

- Add SGP4 propagation from live TLEs
- Add PDF export for mission reports
- Improve mobile layout on small screens
- Add Kessler cascade simulator
- Add historical debris event replay datasets
- Improve 3D orbit performance
- Add satellite pass prediction by location
- Expand Space Traffic Control maneuver scenarios

Suggested labels:

```text
good-first-issue
frontend
orbital-mechanics
data-source
documentation
enhancement
```

## 3. Record A Short Demo

Make a 30-60 second screen recording:

```text
This is OrbitGuard, a space sustainability analyzer.
It visualizes orbital congestion, debris risk, and launch impact.
Users can design satellite missions, simulate launches, and export mission reports.
The goal is to make space sustainability easier to understand.
```

Show these screens:

1. Homepage
2. Mission Design Studio
3. 3D orbit dashboard
4. Launch simulator
5. Space Traffic Control Center

## 4. Share It For Feedback

Use this post:

```text
I am a high school student interested in aerospace engineering, and I built OrbitGuard: a web app that visualizes space debris, simulates satellite launch impact, and helps users compare sustainable mission designs.

Live demo: https://orbitguard.vercel.app
GitHub: https://github.com/harshithpr/orbitguard

I would appreciate feedback on the aerospace modeling, UI, data sources, and roadmap.
```

Good places to post:

- LinkedIn
- r/aerospace
- r/space
- r/webdev
- r/learnprogramming
- r/dataisbeautiful
- school STEM clubs
- robotics/aerospace Discord communities

## 5. Keep It Alive

Commit small improvements weekly:

- Improve README screenshots
- Fix one mobile layout issue
- Add one API example
- Add one test
- Improve one data visualization
- Add one roadmap issue
- Update screenshots after major UI changes

Popular repositories look useful, polished, and active. The best strategy is to keep OrbitGuard as your main public project and improve it consistently.
